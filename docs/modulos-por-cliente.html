ISEO RH — Módulos por cliente

Propuesta para configurar qué secciones usa cada empresa · Julio 2026

1. Situación actual

ISEO RH es una plataforma multi-empresa. Cada cliente (empresa) tiene sus colaboradores y usuarios con distintos roles (Admin RRHH, Supervisor, Empleado).

Hoy todas las empresas ven el mismo menú de secciones. Lo que se puede hacer es activar o desactivar módulos por cada empresa: una puede tener el paquete completo y otra solo lo que necesita, sin perder datos ni rearmar la plataforma.

2. Módulos base Recomendado siempre

Sin estos módulos la plataforma casi no se puede operar.

Módulo Para qué sirve
InicioResumen del día según el rol del usuario
Colaboradores / Mi legajoAltas, fichas, documentos, checklist, bajas (se conserva el historial)
PermisosInvitar usuarios y asignar roles
ConfiguraciónDatos de la empresa, horarios, tolerancia, etc.
AyudaPreguntas frecuentes y asistencia

3. Módulos opcionales

Módulo Para qué sirve Suele ir con…
Ausencias Vacaciones, licencias, home office, carga manual, export a Excel Independiente
Fichaje Ingreso/egreso, kiosco, carga manual Turnos y Reportes
Turnos Horarios vs. fichada (tarde, salida anticipada) Fichaje
Reportes Ausentismo, tardes, extras, presentismo Fichaje (+ Turnos)
Recibos Carga de PDF, firma empleador/empleado, carga masiva Independiente o con Remuneraciones
Remuneraciones Brutos, no remunerativos, descuentos, monotributo Independiente o con Recibos
Agenda Eventos de la empresa y cumpleaños Independiente
Comunicaciones Consultas, reclamos y pedidos (centraliza el WhatsApp) Independiente
Documentos a firmar PDF de políticas / reglamentos con firma digital Independiente
Organigrama Quién reporta a quién Independiente
Convenio Texto del CCT y consulta asistida Independiente

Nota: “Empresas” y “Finanzas” son herramientas internas de ISEO (superadmin), no se configuran como módulo del cliente final.

4. Dependencias recomendadas

1. Siempre: módulos base.

2. Paquete “Control horario” todo o nada
Fichaje + Turnos + Reportes.
Si se activa Turnos sin Fichaje, la experiencia queda incompleta.

3. Paquete “Liquidación” flexible
Solo Recibos, solo Remuneraciones, o ambos.

4. El resto, sueltos: Ausencias, Agenda, Comunicaciones, Documentos a firmar, Organigrama, Convenio.

5. Ejemplos de planes (opcional)

Si preferís no elegir módulo por módulo en cada alta, se pueden armar planes:

Plan Incluye
BásicoBase + Ausencias + Agenda
OperacionesBásico + Control horario (Fichaje, Turnos, Reportes)
LiquidaciónBásico + Recibos y/o Remuneraciones
FullTodos los módulos

Alternativa: checklist libre al dar de alta cada empresa (sin planes fijos).

6. Cómo se vería para cada cliente

  • Al crear o editar una empresa, ISEO elige qué módulos quedan activos.
  • El Admin / empleados de esa empresa solo ven esas secciones en el menú.
  • Si un módulo se apaga después, los datos no se borran; solo deja de mostrarse.

Pregunta para definir

¿Preferís planes fijos (Básico / Operaciones / Liquidación / Full) o elegir módulo por módulo en cada alta de cliente?
